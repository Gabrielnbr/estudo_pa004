Notação

1.2.3 - nome - ação

1 = Em que ordem o código deve ser executado.
2 = Em qual cíclo do CRISP este código está.
3 = Qual a versão temporária do código. Caso esteja em 'a' é a versão consolidáda do código para 
nome = nome do autor
ação = ação do autor no código

Ex.:
1.1.3-gabriel-coleta-e-descrição-dados
É o primeiro passo de execução código, na primeira versão do cíclo do CRISP,
sendo a 3º versão salva daquele cíclo do CRISP, pelo autor gabriel, com as ações de coleta e descrição dos dados.

2.3.a-gabriel-coleta-e-descrição-dados
É o segundo passo de execução código, na terceira versão do cíclo do CRISP (já passei por esse ponto 3 vezes),
sendo a versão final desse cíclo do CRISP, pelo autor gabriel, com as ações de featuring engenniring.

Obs.:
Durante o cíclo do CRISP não há nescessidade de ter que repassar por todos os notebooks,
uma vez que ele funcione para as versões seguintes.
